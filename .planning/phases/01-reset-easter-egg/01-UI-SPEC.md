## Interaction Contracts

### Click Counter (Hidden)

- **Target:** `<h1 class="text-h5 mb-6">{{ t('settings.title') }}</h1>` (existing h1 element)
- **Trigger:** Click event on the h1
- **State:** Internal `clickCount` ref (initialized to 0)
- **Behavior:** Increment counter on each click
- **Reset condition:** When modal opens (counter resets to 0)
- **No visual feedback:** Counter state is invisible (hidden easter egg)

### Reset Confirmation Modal

**Structure:** Follow existing import confirmation dialog pattern from Settings.vue (lines 225–234)

```vue
<v-dialog v-model="showResetConfirm" max-width="360">
  <v-card rounded="xl">
    <v-card-text class="pt-6">{{ t('settings.reset_db_desc') }}</v-card-text>
    <v-card-actions>
      <v-spacer />
      <v-btn variant="text" @click="cancelReset">{{ t('common.cancel') }}</v-btn>
      <v-btn 
        color="error" 
        variant="tonal" 
        :loading="resetLoading"
        @click="confirmReset"
      >
        {{ t('settings.reset_confirm') }}
      </v-btn>
    </v-card-actions>
  </v-card>
</v-dialog>
```

**State Management:**

| Ref | Type | Initial | Purpose |
|-----|------|---------|---------|
| `clickCount` | `Ref<number>` | 0 | Click counter for title |
| `showResetConfirm` | `Ref<boolean>` | false | Modal visibility |
| `resetLoading` | `Ref<boolean>` | false | Loading state during request |

**Modal Behavior:**

1. **Open condition:** When `clickCount === 5`, set `showResetConfirm = true` and reset `clickCount = 0`
2. **Cancel:** Close modal (set `showResetConfirm = false`), no side effects
3. **Confirm:** 
   - Set `resetLoading = true`
   - POST to backend reset endpoint (Wayfinder route: `settings.reset`)
   - Wait for response
   - On success: Show snackbar, close modal, set `resetLoading = false`
   - On error: Show error snackbar, close modal, set `resetLoading = false`

### Loading State

- **Target:** Confirm button in modal
- **Indicator:** `:loading="resetLoading"` prop (Vuetify standard)
- **Behavior:** Button shows spinner and disables while `resetLoading` is true
- **Duration:** From request submission until response returns

### Success / Error Feedback

**Pattern:** Reuse existing snackbar structure from Settings.vue (lines 237–239)

```vue
<v-snackbar v-model="snackbar" :color="snackbarColor" timeout="3000" location="bottom">
  {{ snackbarMessage }}
</v-snackbar>
```

**State:**

| Ref | Type | Purpose |
|-----|------|---------|
| `snackbar` | `Ref<boolean>` | Visibility |
| `snackbarMessage` | `Ref<string>` | Dynamic copy |
| `snackbarColor` | `Ref<'success' \| 'error'>` | Color variant |

**Success:** `t('settings.reset_success')` with `color="success"`, timeout 3000ms
**Error:** Backend error message with `color="error"`, timeout 3000ms
